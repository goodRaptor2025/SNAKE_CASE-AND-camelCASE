Cómo nombramos variables, funciones y más

snake_case

El snake_case es una convención de escritura donde todas las palabras se escriben en minúsculas y se separan mediante un guion bajo (_). Su nombre proviene de la analogía con una serpiente que se desplaza de forma horizontal y baja, similar a cómo luce el texto separado por guiones.

Es ampliamente usado en lenguajes como Python, Ruby y en bases de datos SQL. También es el estándar para nombres de archivos, variables de entorno y claves en APIs REST. Favorece la legibilidad al mantener una apariencia uniforme y plana.

# Python — snake_case
def calcular_precio_total(precio, impuesto):
  precio_con_impuesto = precio * (1 + impuesto)
  return precio_con_impuesto

camelCase

El camelCase es una convención en la que la primera palabra se escribe en minúsculas y cada palabra siguiente comienza con mayúscula, sin espacios ni separadores. Su nombre hace referencia a las jorobas de un camello, que representan las letras mayúsculas intercaladas en el texto.

Es el estándar en lenguajes como JavaScript, Java, TypeScript y Swift. Se utiliza principalmente para nombrar variables, funciones y métodos. Su forma compacta resulta natural en entornos donde la fluidez de lectura en código orientado a objetos es clave.

// JavaScript — camelCase
function calcularPrecioTotal(precio, impuesto) {
  const precioConImpuesto = precio * (1 + impuesto);
  return precioConImpuesto;
}
